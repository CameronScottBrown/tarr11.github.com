A New Year of Coding

Jan 12th, 2014

This week, we started the new year with one new coding class, and the two additional ones. By the end of the week, we had two more classes, including our first girls-only class. So, we’ve gone from 12 coders to almost 30.

So, thank you all for participating and recommending the Mill Valley Code Club to your friends. I know it wouldn’t be here without your support and enthusiasm.

On Using Scratch

When I first started code club, I was not sure which platform to use. I use Code Combat in the first session, which is a fun way to get kids excited about coding.

However, I eventually chose Scratch as a tool to use, since it seemed the most mature tool. I had been hearing good things about Tynker, but I didn’t choose it initially because it was going to add costs to the class, and I needed to learn for myself what the curriculum was going to be like.

The first mvcoders I had join were mostly an outgrowth of Elijah’s soccer team – more than half the kids came from that group. More than half of them liked Minecraft. Since it was a bit more uniform, it made sense to teach the same curriculum to all of them. Our goal in that class was to make a video game. This resonated with most of the kids as that was something that they wanted to do.

The intro class was geared towards getting kids to understand the Scratch platform, and also to start thinking about solving problems in a new way. Coding requires you to plan ahead, and by the end of the first session, most of the kids had figured out how to orchestrate sophisticated games and anmiations.

Jumping

I wanted to have the Wednesday kids explore some basic physics motions so that their games could feel more realistic. Turns out, teaching physics to 4th and 5th graders is not so easy! They still haven’t mastered negative numbers and coordinates, and they haven’t been much introduced to Physics concepts in class yet. So, we spent some time talking about speed, velocity and acceleration. We learned that when you jump, your speed goes to zero at the top of your jump.

These concepts are not easy; it may take a few sessions to get the hang of it. However, if they can master jumping in their code, they will be ready for more advanced Math and Science classes at school.

Different Strokes

As I’ve watched the kids learn to code, I’ve noticed that there are several different learning styles that the kids fall into. I’ve been trying to create a program that allows each of these types of kids to have a good experience.

The Curious Explorer

These kids love to try new things. They are always pushing the platform to it’s limits, kicking the tires, and seeing what it can do. These are the kids who not only have tried Scratch, but are interested in Javascript, Python, Ruby and even Unity 3D!

The Digital Artist

Some kids come to coding with either an artistic or musical interest. They spend much of their time perfecting animations, creating music, using vector drawing tools and illustrating.

The Hardcore Hacker/Gamer

The hacker and gamer (two separate categories for adults) loves to play. They are adept at taking pieces of things and mashing them together in new ways.

I’d like to cater to each group more specifically. I think it would be fun to try out new tools for our explorers, maybe do some advanced Photoshop, Illustrator (and Garage Band??) for the Artists, and encourage the hackers to dream up new takes on existing games.

Trying out Tynker

Starting this week, I am going to mix in some Tynker sessions as well. The interface to Tynker is not that different from Scratch, and it has some good tools for creating curriculums for kids, while still letting the different “coder-types” have a good time. It will also make it easier to create coding curriculums that satisifies kids with different interests and who work at different paces.

Socializing and Coding

Collaboration and socializing are an important part of code club. I had the Monday class wear headphones the whole time. I did this because some of the Wednesday classes had gotten too noisy, and I wanted to cut that down a bit. However, the kids in the Monday class were simply too quiet! So, I think this week, we’ll probably have headphones, but mainly for when we need to concentrate.

Debugging and Unblocking

I see one of my main jobs in code club as “Chief Debugger”. Perhaps your child is doing some coding at home; it can be tough when they run into problems and you don’t really have the time or knowledge to help them solve them.

A key value of code club is helping kids get past roadblocks. Every software package has bugs and issues that make it less than perfect. For new coders, hitting a roadblock in your coding adventure can be incredibly frustrating. There’s so much to learn, and you are excited about it, but trying to figure out how to solve technical problems can be time consuming and require a lot of patience.

Summer Code Camp

I’ve heard from several parents that they are interested in a Marin Code Camp this summer for their kids. I’m considering doing that. However, I doubt that I would run “full-day” sessions as I don’t think it’s appropriate for kids this age to be on the computer all day long. However, I think, because they won’t be in school, that having slightly longer sessions might make sense. Please let me know if you are interested or have any thoughts on it.

Thoughts and Comments?

For many parents, code club is a new experience. I welcome your comments and feedback either here or by emailing me directly.
